The first-ever decentralized data storage platform powered by IPFS and heavily secured by using smart contracts
Freak Cloud moves data transfers and data storage off-chain through IPFS, combined with an on-chain protocol achieved through cryptographic proofs and an incentive model to ensure quality, security and transparency of work of data storage and data transfer.

Data storage and data transfer are done through IPFS in Freak Cloud, which is highly efficient and cheap to use while being completely decentralized. Thanks to IPFS, Freak Cloud also benefits from its wide ecosystem of community-built tools and plugins. Freak Cloud has a data contract system which combines with IPFS, incentivizing the providers, ensuring quality of work from providers, and guaranteeing the best user experience in a transparent way. Freak Cloud integrates a unique Merkle-tree-based proof of storage protocol, ensuring that data is properly stored, also significantly reducing verification cost and proof sizes. Freak Cloud is also powerful in that it has a smart incentive model, which properly incentivizes providers for both data storage and up-time, encouraging providers to give out the best services and securing the network.

Special Technology of Freak Cloud
EVM-based decentralized application
Freak Cloud can be deployed on an EVM environment of Near blockchain, inheriting security, decentralization, and scalability of its. In return, users and dapps on Near can conveniently use the data storage service of Freak Cloud.
Merkle-tree-based proof of data storage system
Verification in Merkle proof can be done in log(n) of computational cost and data size while n is those of normal direct verification. For example, if a direct verification normally needs 1000000 instructions, Merkle proof just needs log(1000000) = 14 instructions.
Anti-fraud feature of Smart incentivization model
A smart incentivization model eliminates frauds and secures the network. Notably, Freak Cloud easily handles a case that many decentralized data storage can not solve: the data provider might store the data and is able to submit data storage proofs while not sending the data to the user.
